# 음악 이론 기초

## 음(音)과 피치

음악의 가장 작은 단위는 **음(note)**. 음은 **피치(pitch)** — 소리의 높낮이 — 로 구분.

서양 음악은 피치를 **12개의 음**으로 나눔. 이 12개가 반복되며 옥타브 상승:

```
C  C#  D  D#  E  F  F#  G  G#  A  A#  B  | C(한 옥타브 위)
도 도# 레 레# 미 파 파# 솔 솔# 라 라# 시
```

- **반음(semitone)**: 인접한 두 음 사이의 간격 (C→C# = 반음 1개)
- **온음(whole tone)**: 반음 2개 (C→D = 온음 1개)

온음은 반음 2개의 편의 단위. 장음계 패턴 표현 시 사용.

C가 기준인 이유: 흰 건반만 눌렀을 때 검은 건반(#/b) 없이 장음계가 완성되는 유일한 음 → 교육 기준.

서양 음악 전체(장조·단조·코드)는 이 12개 음의 조합.

## 음계(Scale)

12개 음 중 특정 패턴으로 골라낸 음들의 집합.

**장음계(major scale) 패턴:** `온 온 반 온 온 온 반`

C 장음계:
```
C → D → E → F → G → A → B → C
  온   온  반  온   온   온  반
= 도  레  미  파  솔  라  시  도
```

다른 음에서 시작해도 같은 패턴 적용 → 장음계. G 장음계:
```
G → A → B → C → D → E → F# → G
  온   온  반  온   온   온   반
```
→ 패턴 유지 위해 F → F#으로 상승. 악보에서 #(샵)이 붙는 이유.

## 단음계(Minor Scale)

**자연 단음계 패턴:** `온 반 온 온 반 온 온`

A 단음계:
```
A → B → C → D → E → F → G → A
  온  반  온   온  반  온   온
= 라  시  도  레  미  파  솔  라
```

A 단음계 = C 장음계와 **같은 음 7개**, 시작점만 다름.

| | 시작점 | 느낌 |
|--|--|--|
| C 장음계 | C(도) | 밝음 |
| A 단음계 | A(라) | 어두움 |

느낌이 다른 이유: **중심음(home)** 이 다름. 장음계는 C로 돌아올 때 안정, 단음계는 A로 돌아올 때 안정.

AI 프롬프팅 `minor key` = 어두운·긴장된 분위기, `major key` = 밝은·긍정적 분위기의 근거.

## 코드(Chord)

여러 음을 동시에 내는 것. 기본 단위는 **트라이어드(triad)** — 음계의 1·3·5번째 음 동시 사용.

C 코드 = C(1) + E(3) + G(5)

| 코드 | 구성 | 느낌 |
|------|------|------|
| C major | C + E + G | 밝음 |
| C minor | C + E♭ + G | 어두움 |

차이: 가운데 음(3음) 하나. E → E♭(반음 낮춤) = 단코드.

코드 저작권: 코드 진행 자체는 저작권 보호 안 됨. 표절 시비는 멜로디·리듬·편곡 텍스처 유사성에서 발생.

Phonk 장르 기준 필요 코드: `Am`, `Dm`, `Em`, `G`, `F`, `Bdim` 6~8개로 루프 80% 커버.

**외워야 하는 것:** 12개 음 이름, 장·단음계 패턴, 자주 쓰는 코드 10개.
**이해로 충분:** 코드 구성 원리(1·3·5음), 코드 진행, 스케일 응용.
프로듀서·AI 프롬프터는 암기보다 **원리 이해 + 귀 훈련**이 핵심.

## 코드 진행(Chord Progression)

코드를 순서대로 나열한 것. 곡의 감정 흐름 뼈대.

**로마 숫자 표기:** 음계의 몇 번째 음을 뿌리로 삼는지 표시.
```
C 장음계:  C   D   E   F   G   A   B
위치:      I   II  III IV  V   VI  VII
```
소문자(i, vi) = 단코드, 대문자(I, V) = 장코드.

**I-V-VI-IV** (팝의 기본): C 장조 기준 `C - G - Am - F`

**i-VII-VI-VII** (Phonk/단조): A 단조 기준 `Am - G - F - G`

로마 숫자로 쓰는 이유: 키가 바뀌어도 진행의 **느낌**이 동일. `i-VI-III-VII`은 어느 단조에서도 같은 긴장감.

## BPM과 박자(Time Signature)

**BPM (Beats Per Minute):** 1분에 박자가 몇 번 치는지. 템포 단위.

**박(beat):** 음악 기본 단위 펄스. 발로 리듬 맞출 때 발이 닿는 순간 하나하나.
**박자(time signature):** 한 마디에 박이 몇 개 묶이는지 규칙.

```
4/4 박자: | 박 박 박 박 | 박 박 박 박 |  → 4개씩 묶음
```

팝·힙합·EDM·Phonk 대부분 4/4.

**1마디 길이 계산 (4/4 기준):**
```
1박 = 60초 ÷ BPM
1마디 = 1박 × 4
```

| BPM | 1마디 |
|-----|-------|
| 80  | 3.0초 |
| 128 | 1.875초 |
| 150 | 1.6초 |

Phonk BPM 범위: 130~160 BPM. Dark Phonk는 80~100 BPM + 하프타임 드럼.

**순서:** BPM 결정 → 박자 결정 → 마디 길이 자동 결정. Phonk/EDM은 BPM 고정 (루프 기반이라 변하면 루프 깨짐).

**DAW:** 음악 제작 소프트웨어. Phonk = FL Studio 주류. Suno 생성 → FL Studio 편집 워크플로우도 일반적.

## 키(Key)

곡이 중심으로 삼는 음 + 음계. "A minor 키" = 중심음 A, 사용 음 A B C D E F G 고정.

같은 코드 진행도 키에 따라 음높이 달라짐:
```
i-VII-VI-VII in A minor: Am - G - F - G
i-VII-VI-VII in D minor: Dm - C - Bb - C
```

Phonk 흔한 키:
| 키 | 느낌 |
|----|------|
| A minor | 기본, 흰 건반만 |
| D minor | 더 무겁고 어두움 |
| F# minor | 날카롭고 공격적 |

major/minor 패턴은 고정:
- **major:** `온온반온온온반` = `2·2·1·2·2·2·1`
- **minor:** `온반온온반온온` = `2·1·2·2·1·2·2`

시작음이 흰 건반이든 검은 건반이든 패턴 동일 적용. 검은 건반 시작 예:
```
F# major: F# G# A# B C# D# E# F#
```

major/minor는 검은/흰 건반 차이가 아님 — **3음이 반음 낮으면 minor, 그대로면 major**.

단음계 변형: 자연(기본)·화성(7음 반음 올림)·선율(6·7음 변형). Phonk 프롬프팅에서 `minor` = 자연 단음계 기준.

Phonk에서 `phrygian scale` 프롬프트도 가능 — minor보다 더 어둡고 공격적.

## 멜로디(Melody)

음들이 시간 순서대로 나열된 것. 코드가 세로(동시)라면 멜로디는 가로(순서).

```
코드:    C + E + G  (동시에)
멜로디:  C → E → G → E → C  (순서대로)
```

구성 요소:
1. **피치** — 어떤 음을 쓰냐 (키/음계 안에서 선택)
2. **리듬** — 각 음을 얼마나 길게·짧게 배치하냐

Phonk 멜로디 특징:
- 반복적·최면적 패턴
- 음 수 적음 (3~5개로 루프)
- 낮은 음역대 선호
- 샘플(옛날 음악 조각)을 멜로디로 쓰는 경우 많음

Suno 프롬프팅:
```
repetitive melodic hook       # 반복 멜로디 루프
minor pentatonic melody       # 단조 5음계 멜로디
haunting melody               # 귀신 들린 듯한 멜로디
low-pitched synth lead        # 낮은 음역 신스 멜로디
```

멜로디·코드 진행은 같은 키 안에서 움직여 자연스럽게 어울림.

## 텍스처(Texture)와 레이어링

텍스처 = 동시에 겹치는 소리의 밀도. 주관적·장르 상대적 개념.

| 텍스처 | 설명 |
|--------|------|
| Thin/Sparse | 적은 레이어, 공간감 |
| Full/Layered | 많은 레이어, 꽉 찬 느낌 |
| Dense | 매우 촘촘 |

Phonk 기본 레이어 구조:
```
1. Sub bass    — 매우 낮은 저음 (느껴지는 저음)
2. 808 bass    — 중간 베이스 (Phonk 정체성)
3. Kick drum
4. Hi-hat roll
5. Synth melody
6. Dark pad    — 배경 분위기
```

텍스처 형용사보다 악기 직접 명시가 더 정확:
```
kick + 808 bass + rolling hi-hats + synth melody + dark pad
```

Suno 프롬프팅: `full texture` / `sparse` / `minimal` / `layered synths`

## 텐션(Tension)과 해결(Resolution)

**텐션** = 불안정한 느낌, 해결 기다리는 상태.
**해결** = 안정으로 돌아오는 느낌.

```
Am - G - F → G(텐션) → Am(해결)
```

Phonk = 해결을 의도적으로 미루거나 생략 → 지속적 긴장감·최면적 그루브.

```
i-VII-VI-VII 반복 → Am으로 강하게 복귀 없음 → 계속 떠도는 느낌
```

마지막 코드가 i(Am)로 끝나면 안정(팝), VII(G)로 끝나면 허공에 뜬 느낌(Phonk).

Suno 프롬프팅:
```
unresolved tension
hypnotic loop, no resolution
dark, tense atmosphere
```

## 리듬과 그루브

박자(4/4)는 틀, **리듬**은 그 틀 안에서 소리 배치.

```
4/4 마디:  1    2    3    4
           강   약   중   약
```

**드럼 세트 핵심:**
| 악기 | 소리 | 역할 |
|------|------|------|
| 킥(Kick) | 쿵 | 강한 박 (1·3박) |
| 스네어(Snare) | 딱 | 엇박 (2·4박) |
| 하이햇(Hi-hat) | 치치치 | 세분화 리듬 |

하이햇 = 두 심벌즈를 겹쳐놓은 것. 닫힘=`치`, 열림=`츠~`.
Phonk 특유의 빠른 `치치치치` = 롤링 하이햇 (트랩 유래).

**그루브:** 리듬이 정박에서 살짝 어긋나는 것 → 인간적 느낌.
- Quantize = 격자에 정확히 맞춤 (기계적)
- Swing/Groove = 의도적으로 밀거나 당김

Suno 프롬프팅:
```
rolling hi-hats          # Phonk/Trap 패턴
tight quantized drums    # 기계적·딱딱
swinging hi-hats         # 그루브감
loose, humanized rhythm  # 인간적 느낌
```
