## 액션 동사(Action Verb) — "책임졌다"가 아니라 "무엇을 했다"

**약한 패턴 — 책임·역할만 서술:**
```
Responsible for managing a team.
In charge of customer service.
Worked on a marketing project.
```
"responsible for", "in charge of", "worked on"은 역할을 말할 뿐 행동을 말하지 않음.

**강한 패턴 — 액션 동사로 시작:**
```
Managed a team of 5.
Led customer service operations.
Launched a marketing campaign.
```
동사 하나로 바로 행동이 드러남.

**액션 동사 카테고리별 예시:**
```
리더십:    led / directed / managed / supervised / coordinated
성과:      achieved / improved / increased / reduced / generated
창조:      built / designed / developed / created / launched
분석:      analyzed / evaluated / researched / identified
```

**핵심 규칙:** 이력서 문장은 거의 항상 동사로 시작. "Responsible for X" 구조를 피하고 X를 행동 동사로 바꿔 문장을 새로 시작.
```
❌ Responsible for increasing sales.
✅ Increased sales by developing a new outreach strategy.
```

한국어 이력서의 "~업무를 담당함"이 전형적인 약한 패턴 — 영어 이력서는 "담당함" 대신 "무엇을 했는가"로 시작하는 게 기본.

## STAR 기법 — 성과를 구조적으로 서술하기

STAR = Situation(상황) → Task(과제) → Action(행동) → Result(결과). 자소서·면접 답변에서 왜·어떻게·얼마나 잘했는지를 구조적으로 보여주는 프레임워크.

**네 단계:**
```
Situation:  어떤 배경·문제 상황이었나
Task:       내가 맡은 구체적 과제는 무엇이었나
Action:     내가 실제로 취한 행동은 무엇이었나
Result:     그 결과 무엇이 달라졌나 (숫자로)
```

**예시 — "팀워크 경험을 말해달라" 질문에:**
```
S: Our team was falling behind on a product launch deadline.
T: I was responsible for coordinating between design and engineering.
A: I set up daily 15-minute sync meetings and created a shared tracker.
R: We launched on time and reduced miscommunication by 40%.
```

**자소서에서 STAR 압축 버전:**
이력서는 STAR를 한 줄로 압축(Action+Result 중심), 자소서는 STAR를 문단으로 풀어서 스토리텔링.
```
이력서: Reduced miscommunication by 40% by implementing daily sync meetings.
자소서: (S-T-A-R 네 문장으로 풀어서 서술)
```

**핵심 원리:** 액션 동사가 STAR의 "A(Action)" 부분에 그대로 들어감 — 액션 동사 + STAR 구조가 합쳐지면 이력서·자소서 문장의 기본 골격이 완성됨.

**흔한 실수 — Result 생략:**
```
❌ I coordinated between design and engineering teams.
✅ I coordinated between design and engineering teams, cutting delays by 40%.
```
Result 없이 끝내면 "뭘 했는지"는 알아도 "그래서 뭐가 좋아졌는지"가 안 보임 — 채용 담당자가 가장 궁금해하는 부분.
