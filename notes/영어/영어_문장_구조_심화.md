## 정보 구조 — 화제(Topic)와 초점(Focus)

화제(Topic) = 문장이 무엇에 대한 이야기인지. 보통 주어 자리.
초점(Focus) = 새로운 정보·강조 포인트. 보통 문장 끝 또는 강세.

- The dog bit the man. → 화제: 개 / 초점: bit the man
- The man was bitten by the dog. → 화제: 남자 / 초점: was bitten

구정보(Given) vs 신정보(New): 구정보는 앞, 신정보는 뒤.
- A dog attacked someone. The dog was later caught.
  (The dog = 구정보 / was later caught = 신정보)

---

## 문말 초점 (End-Focus) 원칙

중요한 정보·새 정보·긴 요소는 문장 끝으로.

- 가주어 it으로 that절을 뒤로:
  어색: That she passed surprised everyone.
  자연: It surprised everyone that she passed.

- 짧은 것 앞, 긴 것 뒤:
  어색: She gave the book she found in the attic to John.
  자연: She gave John the book she found in the attic.

어색한 영어 문장의 원인 대부분 = 이 원칙 위반.

---

## 문두 전치 (Fronting / Topicalization)

보통 뒤에 있어야 할 요소를 앞으로 끌어내 화제·대조 강조.

- 목적어 전치: I like coffee. Tea, I can't stand.
- 대조 전치: Some people love it. Others, not so much.

한국어 `은/는` = 화제 표시 조사. 영어는 동등한 역할을 어순 이동으로:
- 커피는 좋아해. 차는 못 마셔.
- Coffee I like. Tea I can't stand.

## 병렬 구조 (Parallel Structure)

접속사로 연결할 때 같은 문법 형태끼리.

❌ She likes reading, to swim, and going to the gym.
✅ She likes reading, swimming, and going to the gym.

❌ He's smart, hardworking, and has a sense of humor.
✅ He's smart, hardworking, and funny. (형용사·형용사·형용사)

❌ She not only sings but also dancing.
✅ She not only sings but also dances. (상관접속사 양쪽 동일 형태)

and/but/or, not only~but also, both~and, either~or 양쪽 → 같은 품사·형태.

## 응집성 장치 — 문장을 자연스럽게 잇기

1. **대명사 참조**: John came in. He looked tired. (He = John)
2. **대용(Substitution)**: "Will you come?" "I hope so." / I need a pen. Do you have one?
3. **생략(Ellipsis)**: She can sing and (she can) dance.
4. **어휘 연결**: They bought a car. The vehicle was red. (car → vehicle 동의어)
5. **접속 장치**: however / therefore / moreover / as a result → 논리 관계 명시

글쓰기: 대명사·생략으로 반복 제거, 어휘 연결로 주제 이어가기, 접속 장치로 논리 흐름 표시.

## 수동태의 전략적 사용

수동태 = 단순 피동이 아닌 **화제 관리 도구**.

**행위자 생략:**
```
Mistakes were made.          (누가 실수했는지 의도적 생략)
The building was demolished.  (행위자 비중요)
```

**화제(Topic) 유지 — 앞 문장 주어 이어받기:**
```
Police arrested the suspect. The suspect was later released.
                              ↑ 수동태로 "the suspect" 화제 유지
```
능동 쓰면: "Police later released him." → 화제가 police로 전환.

**논리적 주어를 앞에 두기:**
```
The data were analyzed using three methods.
→ 행위자(연구자)보다 데이터가 화제 → 수동이 자연스러움
```

**by구 생략 기준:**
- 행위자 불명·비중요·당연 → 생략
- 행위자가 중요한 신정보 → by와 함께 명시
```
The window was broken.            (누가 했는지 불명)
The window was broken by the child. (행위자가 중요 정보)
```

## 문장 길이와 리듬

문장 길이 변화 = 독자 집중을 조율하는 도구.

**짧은 문장 = 충격·강조·결론:**
```
He had everything. Fame. Money. A perfect family. He was miserable.
```
긴 서술 뒤 갑작스러운 단문 → 최대 강조 효과.

**긴 문장 = 복잡한 관계·흐름·배경:**
```
Although the committee had met several times over the course of
the summer, they had yet to reach a consensus on the proposal.
```

**리듬 패턴 — 짧-짧-긴 또는 긴-짧:**
```
She ran. She fell. But she got up and kept going until she crossed
the finish line despite everything working against her.
   ↑단  ↑단                  ↑긴 문장 → 끝에 무게감
```

**단조로운 글 원인:**
- 모든 문장 비슷한 길이 → 리듬 없음
- 모든 문장 같은 구조(주어+동사+목적어) → 기계적

**실용 규칙:**
- 핵심 메시지 → 짧게
- 배경·설명 → 길게
- 3~4문장마다 길이 변화 의식적으로 줌

## 명사화 (Nominalization)

동사·형용사를 명사로 바꾸는 것. 학술·공식 문체의 핵심 특징.

**기본 변환:**
```
decide → decision
develop → development
analyze → analysis
significant → significance
resistant → resistance
```

**동사 문장 vs 명사화 문장:**
```
동사형:  We decided to expand the program.
명사화:  Our decision to expand the program was made carefully.

동사형:  The economy grew rapidly.
명사화:  Rapid growth of the economy occurred.
```

**언제 쓰나:**
- 학술 논문·보고서 → 행위보다 결과·상태 강조할 때
- 화제로 삼고 싶을 때: "The investigation..." (동사 investigate를 화제로 끌어올리기)
- 여러 행위를 하나로 묶을 때: "The collection and analysis of data..."

**과도한 명사화의 함정:**
```
❌ The implementation of the recommendation resulted in the improvement of performance.
✅ Implementing the recommendation improved performance.
```
긴 명사구 연속 → 읽기 어려움. 일상 글쓰기에서는 동사형이 명확.

**균형:**
- 공식·학술: 명사화 적극 활용
- 일상·설득: 동사형 유지, 명사화 최소화

## 존재 구문 (Existential There)

`There is/are` = 새 정보를 도입하는 전용 구조.

**기본 형태:**
```
There is a problem.            (단수)
There are three solutions.     (복수)
There was a man waiting outside. (과거)
There seems to be an error.    (준동사 결합)
```

**왜 쓰나 — 화제 도입:**
```
A problem exists. → 이미 '문제'를 알고 있다고 가정
There is a problem. → '문제'를 처음 꺼냄 (신정보 도입)
```

한국어 "~이 있다"와 기능 같음.

**수일치 — there 뒤 명사 기준:**
```
There is a book on the table.
There are books on the table.
```

**There vs It:**
```
There is a reason for this.  → 이유가 존재함 (신정보)
It is the reason I left.     → 그 이유(이미 언급된 것)를 가리킴
```

**관용 표현:**
```
There's no doubt that...       (~임에 의심 없다)
There's no point in -ing       (~해봐야 소용없다)
There's no denying that...     (~을 부정할 수 없다)
```

## 삽입 구문 (Parenthetical Insertions)

문장 중간에 부가 정보를 끼워넣는 구조. 쉼표·대시·괄호로 표시.

**세 가지 표시 방법:**
```
쉼표:  The report, which took three months, was finally published.
대시:  The report — which nobody expected — changed everything.
괄호:  The report (originally 200 pages) was condensed to 50.
```

**뉘앙스 차이:**

| 표시 | 느낌 |
|---|---|
| 쉼표 | 중립, 가장 일반적 |
| 대시 | 강조·극적 효과, 구어적 |
| 괄호 | 부가·참고 정보, 읽어도 그만 |

**삽입 구문의 종류:**
```
동격:   My boss, a former engineer, understood the problem.
관계절: The city, which I visited last year, has changed a lot.
분사구: The dog, wagging its tail, ran toward us.
부사구: She finally, after years of trying, passed the exam.
```

**흐름 유지 규칙:**
삽입 구문 제거해도 문장이 완전해야 함.
```
My boss, a former engineer, understood the problem.
→ 제거: My boss understood the problem. ✅ 완전
```

**과도한 삽입 함정:**
```
❌ The man, who I met at the conference, which was held in Seoul,
   in March, a particularly busy time for everyone, said hello.
```
삽입 겹치면 주절이 묻힘 → 한 문장에 하나가 원칙.

## 구정보·신정보와 관사 연결

정보 구조가 관사 선택과 직결됨.

**핵심 연결:**
```
신정보(처음 등장) → a/an
구정보(이미 언급) → the
```

```
I saw a dog. The dog was barking at a cat. The cat ran away.
           ↑신  ↑구                   ↑신    ↑구
```

**문맥상 당연히 존재하는 것 → the:**
```
Please close the door.    (이 방에 문이 있음은 당연)
She went to the hospital. (병원이 특정됨)
```

**같은 명사, 다른 관사 = 다른 의미:**
```
I love music.       (음악 일반)
I love the music.   (지금 흘러나오는 이 음악)

Life is short.      (삶 일반)
The life he lived was remarkable. (그의 구체적인 삶)
```

정보 구조를 이해하면 관사가 규칙이 아닌 **신호**로 보임.

## 분열문 (Cleft Sentences)

특정 요소를 강조하기 위해 문장을 둘로 쪼개는 구조.

**It-cleft — it is... that 사이에 강조 요소:**
```
원문: She bought the car yesterday.
강조: It was she who bought the car.       (누가)
     It was the car that she bought.      (무엇을)
     It was yesterday that she bought it. (언제)
```

**Wh-cleft (Pseudo-cleft) — What절로 화제 설정:**
```
What I need is a break.
What surprised me was her reply.
```
`What + 주어 + 동사 = 강조 요소` 패턴. 앞부분 화제, 뒷부분 초점.

**역 Wh-cleft:**
```
A break is what I need.  (초점 더 강하게)
```

**정보 구조와의 연결:**
- It-cleft: 구정보(that절) + 신정보(it is 뒤)
- Wh-cleft: 화제 설정(What절) + 초점(= 이후)

**구어 vs 문어:**
```
구어:  What I want to say is this: be careful.
문어:  It is this point that merits further investigation.
```

## 대등 연결 vs 종속 연결 전략

같은 내용을 대등(and/but/or)으로 연결할지, 종속(because/although/when)으로 연결할지 선택이 의미를 바꿈.

**대등 연결 — 두 절이 동등한 비중:**
```
He worked hard, and he succeeded.
She apologized, but he didn't forgive her.
```
두 사건 나란히 제시. 인과관계 명시 안 함.

**종속 연결 — 한 절이 배경·원인·조건:**
```
Because he worked hard, he succeeded.
Although she apologized, he didn't forgive her.
```
주절 = 중요 정보, 종속절 = 배경 정보.

**선택이 의미를 바꾸는 예:**
```
He was tired and he made mistakes.
→ 두 사실 나열. 인과관계 불명확.

Because he was tired, he made mistakes.
→ 피로가 원인임 명시.
```

**문체 전략:**
- 대등: 독자가 관계 추론 → 간결·문학적
- 종속: 논리 명시 → 논리적·학술적
- 종속절 앞 배치 → 배경 먼저, 결론 나중 (긴장감)
- 종속절 뒤 배치 → 결론 먼저, 이유 나중 (직접적)

## 생략과 대용 심화

**VP 생략 — 조동사·do 남기고 본동사 이하 생략:**
```
She can sing, and he can too.
A: "Did you finish?" B: "Yes, I did."
```

**to 부정사 생략 — to만 남김:**
```
A: "Are you going?" B: "I'd like to."
You don't have to if you don't want to.
```

**gapping — 대등절 중간 동사 생략:**
```
John ordered pizza, and Mary, pasta.
```

**비교절 생략:**
```
She runs faster than I (do).
He's taller than his brother (is).
```

**생략 불가 조건:**
```
❌ She likes him more than I.
→ 오해 생길 때(I도? 나보다?) 생략 금지. 명시 필요.
```

**대용 vs 생략:**
- 생략: 요소 자체 없앰
- 대용: `one/ones/so/do so`로 교체
```
I need a pen. Do you have one?  ✅ (대용)
I need a pen. Do you have?      ❌ (생략 — 비문)
```

## 범위 부사 (Scope Adverbs)

부사가 어느 요소에 걸리는가에 따라 문장 의미가 달라짐.

**only의 위치 변화:**
```
Only she told him the truth.   → 그녀만 말했다
She only told him the truth.   → 말하기만 했다
She told only him the truth.   → 그에게만 말했다
She told him only the truth.   → 진실만 말했다
```

**even, just, almost도 동일 패턴:**
```
Even he failed.      → 그마저도 실패했다
He even failed.      → 실패까지 했다
He failed even that. → 그것조차 실패했다
```

**not의 범위 — 쉼표로 해소:**
```
He didn't come because he was sick.
→ (a) 오지 않았다 — 아팠기 때문에
→ (b) 온 이유가 아픔이 아니다 (다른 이유로 옴)

He didn't come, because he was sick. → (a) 확정
```

**실용 규칙:**
범위 부사는 수식하려는 요소 **바로 앞**에. 모호하면 문장 재구성 또는 쉼표로 명시.

## 문장 구조와 격식 수준

같은 내용이라도 구조 선택으로 격식 수준이 달라짐.

| 비격식 | 격식 |
|---|---|
| 짧은 문장, 구동사 | 긴 복합문, 단일 동사 |
| 대등 연결(and/but) | 종속 연결(however/therefore) |
| 능동태 위주 | 수동태·명사화 혼용 |
| 구어 어순 | 문두 전치·도치 |

**같은 내용, 다른 격식:**
```
비격식: We looked into the problem and found out what went wrong.
격식:   An investigation into the matter revealed the underlying cause.
```

**문체 선택 기준:**
- 이메일·대화 → 비격식 OK
- 보고서·논문·공문 → 격식 구조
- 뉴스·설명문 → 중간

**함정:**
```
❌ The implementation of the solution to the identified problem
   resulted in the achievement of the desired outcome.
✅ Fixing the bug achieved the goal.
```
격식 ≠ 복잡함. 명확함이 우선.

## 주제문과 단락 구조

문장을 넘어 단락 수준의 구조.

**단락 3요소:**
```
1. 주제문 (Topic Sentence) — 단락이 무엇에 관한지
2. 뒷받침 문장 — 근거·예시·설명
3. 마무리 문장 — 요약 또는 다음 단락 연결
```

**주제문 위치:**
- 앞(가장 일반적): 주제문 → 세부 설명
- 문미: 세부 설명 → 주제문 (긴장감·결론 강조)
- 생략: 독자가 추론하는 함축적 글

**구정보→신정보 원칙이 단락에도 적용:**
```
단락 첫 문장 = 앞 단락과 연결(구정보) + 이 단락 화제 도입(신정보)
```

**단락 간 연결 — 연결어 vs 어휘 연결:**
```
연결어:   However, the results showed otherwise.
어휘 연결: The experiment failed. This failure revealed...
           (failure = 앞 문장 요약 → 화제 이어받기)
```

**단락 길이:**
- 하나의 주제 = 하나의 단락
- 1~2문장 → 다른 단락에 합치거나 더 설명
- 10문장+ → 하위 주제로 분리
