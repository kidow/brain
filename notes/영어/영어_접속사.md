## 접속사란 — 단어·절·문장을 잇는 연결고리

두 요소를 연결하는 품사.

| 종류 | 역할 | 예 |
|---|---|---|
| **등위접속사** | 대등한 두 요소 연결 | and, but, or, so |
| **종속접속사** | 주절에 종속절을 붙임 | although, because, when, if |

등위 = 양쪽이 같은 무게.
종속 = 한쪽이 다른 쪽에 의존.

---

## 등위접속사 FANBOYS

| 글자 | 접속사 | 뜻 |
|---|---|---|
| F | for | 이유 |
| A | and | 추가 |
| N | nor | 둘 다 아님 |
| B | but | 대조 |
| O | or | 선택 |
| Y | yet | 강한 대조 |
| S | so | 결과 |

두 절 연결 시 앞에 쉼표:
```
I was tired, but I kept working.
She studied, so she passed.
```

---

## 종속접속사 — 부사절 유형별

| 유형 | 접속사 |
|---|---|
| 시간 | when, while, after, before, until, as soon as |
| 이유 | because, since, as |
| 조건 | if, unless, as long as |
| 양보 | although, though, even though, even if |
| 목적 | so that, in order that |
| 결과 | so … that, such … that |
| 대조 | while, whereas |

종속절이 앞 → 쉼표 필요. 뒤 → 쉼표 없음.
```
Because she was tired, she went home.
She went home because she was tired.
```

---

## while vs whereas — 대조 접속사 구분

`while` = 시간(~하는 동안) 또는 대조(~인 반면에, 비격식)
`whereas` = 대조만 (격식·문어체)

```
While I was cooking, she watched TV.      (시간)
While I like coffee, she prefers tea.     (대조 — 비격식)
I like coffee, whereas she prefers tea.   (대조 — 격식)
```

구분: while 앞 절에 진행 동작 → 시간. 상태·의견·사실 → 대조.

---

## 상관접속사 — 쌍으로 쓰는 접속사

| 상관접속사 | 뜻 |
|---|---|
| `both A and B` | A와 B 둘 다 |
| `either A or B` | A 또는 B |
| `neither A nor B` | A도 B도 아님 |
| `not only A but also B` | A뿐만 아니라 B도 |
| `not A but B` | A가 아니라 B |

수일치: 동사는 B(뒤쪽 명사)에 맞춤.
```
Either she or I am right.
Neither he nor they are wrong.
```

---

## 접속부사 — 접속사처럼 보이지만 부사

`however / therefore / moreover / nevertheless / thus` = 접속사 아님. 두 절을 쉼표로 연결 불가.

```
I was tired, however I kept working.   ❌
I was tired. However, I kept working.  ✅
I was tired; however, I kept working.  ✅
```

| 접속부사 | 뜻 | 비고 |
|---|---|---|
| however | 그러나 | |
| therefore | 따라서 | 일반 격식 |
| thus | 따라서 | 고격식·논문 |
| moreover | 게다가 | 단순 추가 |
| furthermore | 더 나아가 | 앞 주장 강화 |
| nevertheless | 그럼에도 불구하고 | |

패턴: `문장. 접속부사, 문장.` 또는 `문장; 접속부사, 문장.`

---

## because vs since vs as — 이유 접속사 구분

| | `because` | `since` | `as` |
|---|---|---|---|
| 강도 | 강함 — 직접 이유 | 중간 | 약함 |
| 이유 | 새 정보 | 이미 알려진 정보 | 이미 알려진 정보 |
| 격식 | 중립 | 격식 | 고격식·문어 |

```
I stayed home because I was sick.    (이유 = 핵심 새 정보)
Since you're here, let's start.      (이미 알려진 배경)
As it was raining, we stayed inside. (당연히 아는 상황)
```

`since`는 시간("~이후로")도 되니 문맥으로 구분.

---

## although vs though vs even though vs even if

| | 특징 |
|---|---|
| `although` | 격식·중립 |
| `though` | 비격식, 문장 끝 가능 |
| `even though` | 사실 강조 |
| `even if` | 가정·불확실 상황 |

핵심 구분:
```
Even though I was tired, I finished.   (실제로 피곤했음 — 사실)
Even if I am tired, I will finish.     (피곤할 수도, 아닐 수도 — 가정)
```

`though`만 문장 끝 가능:
```
It was cold, though.  ✅
```

**기타 양보 표현:**
```
whether or not         Whether or not you agree, we're moving forward.
however/whatever + 복합관계사   However hard it is, don't give up.
```

---

## if vs unless — 조건 접속사

`unless` = `if … not` = "~하지 않으면" (예외 조건)

```
I'll go unless it rains.       = I'll go if it doesn't rain.
Unless you study, you'll fail. = If you don't study, you'll fail.
```

`unless` + 부정 = 이중 부정 → 금지.
```
Unless you don't come.  ❌
If you don't come.       ✅
```

**조건 뉘앙스를 더 세분화하는 접속사:**
```
as long as        ~하는 한 (제한) — Stay as long as you're quiet.
provided (that)    격식 조건 — You can borrow it, provided you return it.
in case            예방 (지금 행동) — Take an umbrella in case it rains.
suppose/supposing  가설 (구어) — Suppose it rains — should we cancel?
```
if는 중립 조건, 나머지는 각각 제한·격식·예방·가설 뉘앙스를 추가.

---

## so that vs so … that — 목적 vs 결과

**`so that`** = 목적 (~하기 위해서)
```
She studied hard so that she could pass.
```

**`so + 형용사/부사 + that`** = 결과 (너무 ~해서 …하다)
```
It was so cold that I stayed home.
```

**`such … that`** = 결과 (명사 앞)
```
It was such a cold day that I stayed home.
```

구분:
- 뒤에 조동사(can/could/will/would) → 목적 `so that`
- 형용사·부사 강조 → 결과 `so … that`
- 명사 앞 → 결과 `such … that`

---

## 명사절 접속사 — that / whether / if

**`that`** — 사실·생각·감정 전달.
```
I know [that she is right].      (목적어, that 생략 가능)
[That he lied] surprised me.     (주어)
```

**`whether / if`** — "~인지 아닌지" (불확실·의문)
```
I don't know whether/if she will come.
```

`whether`만 가능한 경우:
```
Whether she comes or not doesn't matter.  ✅ (주어 자리)
If she comes or not doesn't matter.       ❌
```

---

## as — 다중 용법 접속사

| 용법 | 뜻 | 예 |
|---|---|---|
| 시간 | ~할 때 (동시) | As I walked in, she left. |
| 이유 | ~이므로 (배경) | As it was late, we left. |
| 방식 | ~대로 | Do as I say. |
| 비례 | ~할수록 | As you get older, time flies faster. |

구분: 동시 진행 → 시간 / 알려진 배경 → 이유 / 동작 따라 함 → 방식.

## 접속사 vs 전치사 혼동 쌍

| 의미 | 접속사 (뒤에 절) | 전치사 (뒤에 명사구) |
|------|-----------------|-------------------|
| 양보 | although / though | despite / in spite of |
| 이유 | because / since | because of / due to / owing to |
| 시간 | while | during |

- Although he was tired, he worked. / Despite his tiredness, he worked.
- Because she studied hard, she passed. / Because of her hard work, she passed.
- While he was sleeping, I left. / During his sleep, I left.

실수 주의: despite he was tired ❌ → despite his tiredness ✅

## now that / once / by the time

now that — "이제 ~했으니까" (변화된 상황이 원인)
- Now that you're here, we can start.
- because와 유사하지만 새로운 상황 변화에 초점.

once — "일단 ~하면" (전환점 조건)
- Once you understand it, it's easy.
- after보다 "이 단계를 넘으면"이라는 느낌.

by the time — "~할 때쯤이면 이미" (기준 시점보다 먼저 완료)
- By the time he arrived, she had already left.
- 뒤 절은 완료 시제(had p.p. / will have p.p.) 자연스러움.

---

## 논리 접속부사 — 위치와 구두점

접속부사(however/therefore 등)는 접속사가 아니므로 콤마만으로 두 절 연결 불가.

올바른 연결:
- 마침표 후 새 문장: He was tired. However, he kept working.
- 세미콜론: He was tired; however, he kept working.
- 콤마 오류(comma splice): He was tired, however he kept working. ❌

가장 자연스러운 위치: 새 문장 맨 앞.

| 의미 | 접속부사 |
|------|---------|
| 역접 | however, nevertheless, nonetheless |
| 결과 | therefore, thus, hence, consequently |
| 추가 | moreover, furthermore, in addition |
| 예시 | for example, for instance |
| 환언 | in other words, that is |

---

## lest / for fear that — 부정 목적 접속사

"~하지 않도록". so that ~ not의 격식·문어체 대안.

lest + 동사원형 (should 생략 가능):
- He whispered lest anyone hear him.
- Study hard lest you should fail.

for fear that + might/would/should:
- She saved money for fear that she would lose her job.

lest는 현대 영어에서 매우 드뭄 (문학·법률·고어체). 읽을 때 알아보는 정도로 충분.
일상에선 in case가 자연스러운 대안.

---

## no sooner ~ than / scarcely ~ when

"~하자마자" as soon as의 격식·문어체 버전. 문두에 오면 도치 필수.

구조: No sooner/Scarcely/Hardly + had + S + p.p. ~ than/when + 과거형

- No sooner had she arrived than it started raining.
- Scarcely had he sat down when the phone rang.
- Hardly had I left before it began to snow.

일상에선 as soon as가 자연스러움. 이 패턴은 독해·시험에서 자주 등장.

---

## given (that) / seeing (that) / considering (that)

"~을 감안하면 / ~인 점을 고려하면". 독립 분사구문에서 파생된 접속사.

**given (that):**
- Given that he's new, he's doing well. (절 앞)
- Given the circumstances, it was the best option. (명사구 앞 — 전치사처럼)

**seeing (that) / seeing as:**
- Seeing (that) it's already late, let's stop here.
- Seeing as you're here anyway, you might as well help.
- 구어적. since와 유사하지만 더 일상적.

**considering (that):**
- Considering (that) she just started, the result is impressive.
- given보다 약간 더 중립적.

셋 모두 since/because와 의미 가깝지만 **"감안하면"이라는 평가·판단** 뉘앙스가 추가됨.

## 방식 부사절 — as / as if / as though

**as** — 실제 방식 "~대로, ~처럼"
```
Do it as I showed you.
Leave things as they are.
```

**as if / as though** — 비현실 "마치 ~인 것처럼" (가정법 시제)
```
He talks as if he knew everything.        (사실은 모름 — 2차)
She looked as if she had seen a ghost.    (본 적 없음 — 3차)
```

as vs as if: as = 실제 방식(직설법) / as if = 비현실(가정법)
구어에서 실제 가능성 있을 때 직설법도 허용: It looks as if it's going to rain.

---

## 부사절의 위치 — 앞 vs 뒤

- 뒤(기본): She passed because she studied. → 주절이 핵심, 이유는 부가
- 앞(강조): Because she studied, she passed. → 이유를 먼저, 결과 뒤에

앞에 올 때 쉼표 필수. 뒤에 올 때 쉼표 선택(있으면 약간 강조).
글쓰기: 앞 배치 = 더 논리적·격식적. 구어: 뒤가 더 자연스러움.
