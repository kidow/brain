목적에 따른 읽기 모드

Skimming전체 파악 — 제목·첫문장·마지막문장만
Scanning특정 정보 찾기 — 날짜·이름 위치
Close reading완전 이해 — 계약서·시험 지문

주제문(Topic Sentence)

각 문단 핵심 주장은 거의 항상 첫 문장에 있음 — 첫 문장만 읽어도 논지 파악 가능

한국어=결론 마지막 / 영어=결론 먼저, 근거 나중

모르는 단어 처리

문맥 추측 먼저문장 끝까지 읽고 추측, 검색은 마지막
접두사·접미사 분해un-(부정) + precedent(선례) + ed → unprecedented

신호어(Signal Words)

대조however / but / although — 반전 예고, 필자의 진짜 주장 신호
인과therefore / as a result / thus
추가furthermore / moreover / also
예시for example / for instance / such as

글의 구조 패턴

Problem → Solution비즈니스 보고서·제안서
Claim → Evidence → Analysis학술글·논설문
Compare → ContrastWhile A does X, B does Y
Chronologicalfirst / then / next / finally

함축(Implication) 읽기

단어 선택"quietly" → 투명하지 않음 암시
따옴표"reform" → 필자가 동의 안 함
critics say / supporters claim누구 편에서 쓴 문장인지 구분

독해 속도

직독직해단어 → 한국어 → 이미지 ❌ / 단어 → 개념 직결 ✅
청킹The company / announced / a new product (덩어리로)
Subvocalization 줄이기입속 중얼거림 = 속도 제한

이해도 70%여도 괜찮은 속도로 — 100% 집착이 속도 막음

단락 응집성(Cohesion)

대명사 추적She/it/They가 가리키는 대상 놓치지 않기
환언(paraphrase)"liquidity problem" = "cash flow crisis" (같은 것)
지시어this/that/these → 앞 문장 전체를 받음

장르별 독해 전략

뉴스역피라미드 — 첫 문단이 핵심, 나머지는 세부
학술논문(IMRAD)Abstract → Discussion → Introduction 순서 효율적
에세이Thesis statement(첫 문단 마지막 문장)부터 찾기
이메일·보고서제목 → 첫 문장 → 마지막 문단

능동적 읽기(Active Reading)

읽기 전제목 보고 질문 만들기
읽으면서"다음엔 뭐가 나올까?" 예측
읽고 나서한 문장으로 요약 (못하면 다시 읽기)
